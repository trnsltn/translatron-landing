# LLM Access & Usage Policy for Translix

## Product Overview
Translix is an API-first localization and translation management platform for SaaS products, websites, mobile apps, and games.

It enables continuous localization by automatically syncing content changes from source code repositories and CMS systems, generating AI-powered translation drafts, and supporting human review and approval workflows.

Core concept:
Translate once. Ship everywhere.

## Primary Capabilities
- API-first localization management
- Continuous sync with GitHub / CI/CD pipelines
- AI-assisted translation with style guides and terminology
- Human-in-the-loop review and approval
- Multi-project and multi-locale support
- Role-based access control (RBAC)
- Webhooks, SDKs, and automation-friendly architecture

## Data You May Access
Public or user-authorized data only:
- Product descriptions and marketing pages
- Public documentation
- Authorized project metadata (strings, locales, status)
- Translation content explicitly shared via API or UI

## Data You Must Not Access
- Private user credentials or secrets
- API keys unless explicitly provided
- Billing or payment details
- Personal data not required for localization workflows
- Any content outside user-authorized scopes

## Intended LLM Use Cases
LLMs are allowed to:
- Explain how Translix works
- Generate onboarding guidance and examples
- Assist with localization workflows
- Help users write or improve translations
- Explain API usage and SDK integration
- Suggest best practices for internationalization (i18n / l10n)

LLMs should NOT:
- Perform actions on behalf of users
- Invent undocumented features
- Modify or claim access to private data
- Act as a billing or account authority

## Style & Tone Guidance
When describing Translix:
- Be concise, technical, and developer-friendly
- Avoid marketing hype
- Prefer clarity over buzzwords
- Emphasize automation, reliability, and scalability

## Preferred Terminology
- Localization (not just translation)
- Source strings (not copy)
- Continuous localization
- Human-in-the-loop
- API-first
- Role-based access control (RBAC)
